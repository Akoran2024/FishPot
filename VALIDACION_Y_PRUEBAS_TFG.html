8. VALIDACIÓN Y PRUEBAS

Esta nueva fase tiene como finalidad garantizar la calidad del software creando una serie de casos de pruebas para descubrir errores subyacentes en el sistema construido.

Objetivo

El objetivo principal de la ejecución de las pruebas se dirige a:

  • Descubrir tantos errores como sea posible.
  • Notificar acerca de los riesgos percibidos del Proyecto.
  • Identificar errores funcionales de la aplicación.
  • Evaluar la calidad del producto.
  • Evaluar la calidad técnica del producto y resolver los errores identificados.
  • Cumplir con los requerimientos establecidos.

Herramientas de Validación

Para la validación del proyecto se han seleccionado herramientas que cubren la lógica de negocio, la interfaz de usuario y la calidad del código fuente:

1. PHPUnit

  • Es el framework estándar para realizar pruebas unitarias en PHP, integrado de forma nativa en Laravel. Su objetivo es aislar cada parte del programa (controladores, modelos) y demostrar que funcionan correctamente de forma individual.

2. Laravel Dusk

  • Herramienta de automatización que testea la interfaz de la aplicación simulando el comportamiento real de un usuario en un navegador, verificando flujos críticos como el registro y el proceso de compra.

3. Laravel Pint

  • Es un corrector de estilo de código PHP que asegura que todo el desarrollo siga los estándares PSR-12 y las convenciones de Laravel, garantizando la limpieza y legibilidad del código.

Ejecución de las Pruebas

A continuación se describen los comandos y procedimientos necesarios para ejecutar el plan de pruebas en el entorno de desarrollo:

A. Pruebas de Funcionalidad (PHPUnit)

  • Se ejecutan mediante el comando: php artisan test
  • Este comando lanza las pruebas unitarias y de funcionalidad, proporcionando un reporte inmediato sobre el estado de las rutas, validaciones y acceso a datos.

B. Pruebas de Interfaz (Dusk)

  • Se ejecutan mediante el comando: php artisan dusk
  • Dusk abre una instancia automatizada de Chrome para verificar visualmente que los componentes de Vue.js responden correctamente a la interacción del usuario.

C. Control de Calidad del Código (Pint)

  • Se ejecuta mediante el comando: ./vendor/bin/pint
  • Este comando analiza y corrige automáticamente el estilo del código para mantener la coherencia técnica en todo el proyecto.
