Guía de Despliegue en Producción

Pasos para subir FishPot a un dominio real (ej. www.tudominio.com)

1. Requisitos Previos

  • Dominio comprado: (ej. en Namecheap, GoDaddy, DonDominio).
  • Servidor VPS: Un servidor virtual con Linux (Ubuntu 22.04 o 24.04 recomendado) en proveedores como DigitalOcean, Linode o Hetzner.
  • Acceso SSH: Capacidad para entrar al servidor mediante consola.

2. Configuración del Dominio (DNS)

Debes apuntar tu nombre de dominio a la dirección IP de tu servidor VPS.

  1. Entra al panel de tu registrador de dominios.
  2. Busca la sección DNS Management o Registros DNS.
  3. Crea un registro tipo A:
    • Host: @
    • Valor: [Tu IP del Servidor]
  4. Crea otro registro tipo CNAME:
    • Host: www
    • Valor: tudominio.com

3. Preparación del Servidor (Stack LEMP)

Ejecuta estos comandos en la consola de tu servidor para instalar lo necesario:

sudo apt update && sudo apt upgrade -y
# Instalar Nginx, MySQL y PHP 8.2+
sudo apt install nginx mysql-server php-fpm php-mysql php-xml php-curl php-gd php-zip unzip -y

4. Clonar el Proyecto y Configurar

Descargar código:

cd /var/www
sudo git clone https://github.com/tu-usuario/FishPot.git
sudo chown -R $USER:$USER /var/www/FishPot

Instalar dependencias y generar claves:

cd /var/www/FishPot
composer install --optimize-autoloader --no-dev
cp .env.example .env
php artisan key:generate
IMPORTANTE: Edita el archivo .env con nano .env y cambia APP_ENV=production, APP_DEBUG=false y pon la APP_URL=https://tudominio.com. También configura las credenciales de la base de datos de producción.

Compilar el Frontend (Vue.js):

Esto generará los archivos finales en la carpeta public/build.

npm install
npm run build

5. Configuración de Nginx

Debes decirle a Nginx cómo servir tu aplicación Laravel.

sudo nano /etc/nginx/sites-available/fishpot

Pega este contenido (ajustando el dominio):

server {
    listen 80;
    server_name tudominio.com www.tudominio.com;
    root /var/www/FishPot/public;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    index index.php;
    charset utf-8;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}
# Activar el sitio y reiniciar Nginx
sudo ln -s /etc/nginx/sites-available/fishpot /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl restart nginx

6. Permisos de Carpetas

Laravel necesita escribir en ciertas carpetas:

sudo chown -R www-data:www-data /var/www/FishPot/storage
sudo chown -R www-data:www-data /var/www/FishPot/bootstrap/cache

7. Seguridad SSL (HTTPS Gratis)

Usa Certbot para instalar un certificado SSL de Let's Encrypt:

sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d tudominio.com -d www.tudominio.com

Sigue las instrucciones y elige la opción de Redirigir todo el tráfico a HTTPS.

8. Tareas de Optimización Final

php artisan migrate --force
php artisan config:cache
php artisan route:cache
php artisan view:cache
¡Listo! Tu aplicación FishPot ahora debería ser accesible desde tu dominio con conexión segura HTTPS.
