Requerimientos No Funcionales

Especificaciones de Calidad y Rendimiento - Proyecto FishPot

RNF-01 Rendimiento

  • Tiempo de Carga: La aplicación debe cargar en menos de 1.5 segundos en conexiones de banda ancha.
  • Concurrencia: Capacidad para manejar hasta 100 usuarios simultáneos sin pérdida de rendimiento.
  • Optimización: Las imágenes de productos deben estar optimizadas para web (WebP/JPEG progresivo).

RNF-02 Seguridad

  • Autenticación: Implementación de Laravel Breeze/Sanctum para la gestión segura de usuarios.
  • Cifrado: Uso obligatorio de algoritmo BCrypt para contraseñas.
  • Protección CSRF: Protección activa contra ataques Cross-Site Request Forgery en todos los formularios.

RNF-03 Usabilidad

  • Responsive Design: Interfaz adaptada 100% a móviles, tablets y PCs usando Tailwind CSS.
  • Navegación: El usuario no debe tardar más de 3 clics en encontrar cualquier producto.
  • Feedback: Mensajes claros de éxito/error en acciones de carrito y login.

RNF-04 Mantenibilidad

  • Código Limpio: Seguimiento de estándares PSR-12 para PHP y Guía de Estilo de Vue.
  • Documentación: Uso de migraciones y seeders para replicar la base de datos fácilmente.
  • Modularidad: Separación clara entre la API (Laravel) y el Frontend (Vue.js).
