0. Visión General del Proyecto

FishPot es una plataforma de comercio electrónico End-to-End para el sector pesquero. No solo digitaliza la venta de productos, sino que actúa como un centro de información para el pescador, integrando datos oceanográficos dinámicos y herramientas administrativas de nivel empresarial.

Propósito Comercial

  • Venta de artículos de pesca (Canarias y Península).
  • Gestión de clientes y fidelización.
  • Automatización de facturación legal.

Propósito Técnico

  • Demostración de arquitectura SPA moderna.
  • Integración de pagos seguros (PCI Compliance).
  • Visualización de datos en tiempo real (APIs).

3. Base de Datos: Esquema y Relaciones

FishPot utiliza un motor relacional MySQL con una estructura de 3ª forma normal.

Entidades Principales

  • Users: Soporta múltiples roles (user, admin).
  • Products: Incluye control de inventario y categorización.
  • Orders: Registra la transacción, el payment_intent de Stripe y el estado logístico.
  • OrderItems: Tabla pivote que registra el precio histórico al momento de la compra (crucial para auditoría).

Relaciones (ORM Eloquent)

User --(1:N)--> Order
Order --(1:N)--> OrderItem
Product --(1:N)--> OrderItem

6. Gestión de Estado Global (Pinia)

FishPot utiliza Pinia para centralizar la verdad de la aplicación.

authStore

Maneja el ciclo de vida de la sesión, persistencia del token y roles de usuario.

cartStore

Lógica compleja de carrito:

  • Validación de stock en tiempo real.
  • Cálculo automático de impuestos e IVA.
  • Persistencia en localStorage para recuperación tras cierre de navegador.

7. Integraciones Maestras

Stripe Payment Gateway

Flujo de seguridad: El cliente nunca maneja datos sensibles de tarjeta. Se usa Server-side Redirection para evitar ataques Man-in-the-Middle.

World Tides API

Localización configurada: Puerto de Arrecife / Famara. Parámetros: datum=LAT (Bajamar astronómica mínima), days=7.

9. Manual del Administrador: Control Total

Como administrador, eres responsable del flujo de caja y logística.

Gestión de Pedidos

Cuando un pedido entra, revisa el stock físico. Si todo es correcto, pulsa "Aceptar". Esto generará la orden de preparación en almacén.

Alertas de Stock

El sistema resaltará en rojo cualquier producto con menos de 5 unidades. Es tu señal para realizar pedidos a proveedores.

10. Mantenimiento y Operaciones CLI

Comandos vitales de mantenimiento para el administrador de sistemas:

  • php artisan cache:clear: Resuelve problemas de visualización tras actualizaciones.
  • php artisan queue:work: Procesa el envío de correos electrónicos en segundo plano.
  • php artisan route:list: Auditoría de todos los accesos al sistema.

11. Seguridad y Optimización

  • Protección CSRF: Laravel protege cada formulario contra peticiones maliciosas externas.
  • Hashing Bcrypt: Las contraseñas nunca se guardan en texto plano.
  • Rate Limiting: Evita ataques de fuerza bruta en el login.
  • Lazy Loading: Las imágenes del catálogo se cargan solo cuando son visibles para el usuario, ahorrando datos.
