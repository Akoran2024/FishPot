Arquitectura Integral, Flujo de Datos y Estructura de Proyecto FishPot

7.1 Árbol Completo de Directorios

A continuación se detalla la estructura jerárquica total del proyecto, abarcando desde el arranque del servidor hasta la lógica de cliente.

FishPot/ ├── app/ # Capa de Aplicación (Lógica de Negocio) │ ├── Http/ │ │ ├── Controllers/ # Manejo de peticiones │ │ ├── Middleware/ # Filtros de seguridad (AdminMiddleware) │ │ └── Requests/ # Validaciones de entrada │ ├── Mail/ # Sistema de correos (InvoiceMail) │ ├── Models/ # Modelado de datos y relaciones │ └── Providers/ # Inyección de servicios (AppServiceProvider) ├── bootstrap/ # Inicialización y Caché del Framework ├── config/ # Configuración Global (DB, Mail, Auth) ├── database/ # Ciclo de Vida de Datos │ ├── factories/ # Generación de datos de prueba │ ├── migrations/ # Estructura de la base de datos │ └── seeders/ # Datos maestros de inicio ├── nginx/ # Configuración del servidor web en producción ├── public/ # Recursos accesibles externamente │ └── imagenes/ # Activos gráficos de la plataforma ├── resources/ # Fuentes del sistema │ ├── js/ # SPA en Vue.js 3 │ ├── views/ # Plantillas de Backend (Blade) │ └── css/ # Estilos TailwindCSS ├── routes/ # Definición de rutas (Web, API, Auth) ├── storage/ # Almacenamiento local y logs ├── tests/ # Suite de Pruebas Unitarias y de Integración ├── docker-compose.yml # Orquestación de contenedores └── vite.config.js # Compilación de activos modernos

7.2 Desglose Exhaustivo por Carpetas

1. Lógica de Servidor (app/)

app/Http/Middleware
Seguridad Capas de filtrado. AdminMiddleware.php protege las rutas sensibles verificando el rango del usuario.
app/Mail
Notificaciones InvoiceMail.php define la estructura de las facturas PDF enviadas tras una compra exitosa.
app/Providers
Service Container AppServiceProvider.php es donde se registran servicios y se configura la lógica de arranque global.

2. Configuración del Sistema (config/)

Archivos PHP que centralizan las opciones del entorno:

ArchivoPropósito
database.phpConexión con MySQL, drivers y configuraciones de Redis.
mail.phpConfiguración de SMTP para el envío de correos (Mailtrap/Gmail).
auth.phpDefine guardias y proveedores de usuarios para la seguridad.
services.phpCredenciales para APIs externas (MeteoBlue, Stripe, etc.).

3. Sistema de Rutas (routes/)

Puntos de entrada de la aplicación:

RutaAlcance
web.phpRutas principales que cargan la SPA y gestionan las peticiones AJAX.
auth.phpEndpoints específicos de Fortify/Breeze para login y registro.
console.phpComandos personalizados para la terminal Artisan.

4. Interfaz de Usuario (resources/js/)

Detalle de la arquitectura del Frontend en Vue 3:

resources/js/components
Atómicos Toast.vue es el sistema de avisos emergentes que se utiliza en toda la web.
resources/js/router
Navegación index.js configura las guardas de navegación (BeforeEach) para proteger el área de cliente.
resources/views/emails
Plantillas invoice.blade.php contiene el diseño HTML de los correos electrónicos.

5. Pruebas y Validación (tests/)

Garantizan que el sistema sea robusto y que las actualizaciones no rompan funcionalidades existentes.
ArchivoTipo de Test
Feature/Auth/Verifica que el flujo de registro y login funcione correctamente.
Feature/ProfileTest.phpAsegura que los usuarios puedan modificar sus datos de forma segura.
Unit/ExampleTest.phpPruebas de lógica pequeña y funciones matemáticas.

6. Infraestructura (Archivos Raíz)

FicheroDescripción
DockerfileInstrucciones para construir la imagen de Docker de la aplicación.
docker-compose.ymlOrquestación de servicios (PHP, Nginx, MySQL, Mailpit).
nginx/default.confConfiguración del servidor web para manejar peticiones y archivos estáticos.
tailwind.config.jsConfiguración de la identidad visual (colores náuticos, tipografías).
