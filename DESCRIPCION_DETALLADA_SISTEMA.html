Anexo: Descripción Detallada de los Ficheros del Sistema

En el presente anexo se detallan los ficheros que componen la solución técnica FishPot, especificando su ubicación dentro de la jerarquía del proyecto y su responsabilidad funcional dentro de la arquitectura del software.

1. Capa de Lógica de Negocio y Persistencia (Backend)

Esta sección comprende los archivos desarrollados en PHP bajo el framework Laravel 12, encargados de la gestión de datos, seguridad y comunicación con servicios externos.

1.1 Controladores (app/Http/Controllers)

StripeController.php Gestiona la integración con la API de pagos Stripe. Implementa los métodos checkout para crear sesiones de pago seguras y success para procesar la confirmación del pedido, actualizando el inventario y persistiendo la transacción en la base de datos.
TideController.php Actúa como puente entre la aplicación y el servicio WorldTides. Obtiene datos meteorológicos y de mareas en tiempo real para las coordenadas de Lanzarote, incluyendo lógica de manejo de errores y datos de respaldo (fallback).
ProductController.php Centraliza la gestión del catálogo. Permite la recuperación de productos con filtros dinámicos y gestiona las operaciones CRUD para el panel administrativo.
OrderController.php Administra el ciclo de vida de los pedidos, desde su creación hasta la actualización de estados (aceptado, enviado, cancelado).

1.2 Modelos de Datos (app/Models)

Product.php Define la entidad Producto. Contiene la lógica para la gestión de precios, stock y la asociación con las imágenes almacenadas en el sistema.
Order.php y OrderItem.php Representan la cabecera y el detalle de los pedidos realizados, estableciendo una relación de uno a muchos (1:N) para desglosar los productos adquiridos.
Client.php Almacena la información extendida de facturación y envío de los usuarios registrados.

2. Capa de Presentación e Interacción (Frontend)

Ficheros desarrollados en Vue.js 3, encargados de proporcionar una experiencia de usuario reactiva y fluida como Single Page Application (SPA).

2.1 Vistas y Páginas (resources/js/views)

Shop.vue Interfaz principal del comercio electrónico. Implementa componentes reactivos para la visualización del catálogo y filtrado dinámico de artículos de pesca.
Cart.vue Gestiona el flujo de compra. Permite al usuario interactuar con el carrito, calcular costes de envío y validar el resumen antes del pago.
Tides.vue Componente especializado que renderiza los datos de mareas, proporcionando una herramienta visual para la planificación de la actividad pesquera.
MyOrders.vue Área personal del cliente que permite consultar el histórico de compras y descargar facturas en formato PDF.

2.2 Gestión de Estado y Navegación

stores/cart.js Implementación de Pinia para la gestión del carrito de compras. Garantiza la persistencia de los datos entre diferentes rutas de la aplicación.
router/index.js Define el mapa de navegación del sistema, gestionando las rutas públicas y las protegidas (admin) mediante guardias de navegación.

3. Configuración y Entorno

.env Fichero crítico de configuración de entorno. Almacena claves de API, credenciales de base de datos y parámetros de servidor específicos del despliegue.
vite.config.js Configuración del motor de compilación Vite. Optimiza el empaquetado de componentes Vue y estilos Tailwind CSS para su despliegue en producción.
database/migrations/ Conjunto de scripts que definen el esquema relacional de la base de datos, permitiendo la portabilidad y el control de versiones de la estructura de datos.
