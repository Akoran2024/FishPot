8. Memoria Técnica del Sistema

8.1 Estructura del proyecto

La carpeta del Proyecto que ha sido denominada como “FishPot” contiene en su interior seis directorios principales: app, bootstrap, config, database, public, resources, routes y tests.

Ilustración 8.1.1: Estructura de directorios principal del proyecto FishPot
  • La carpeta “app” constituye el núcleo de la aplicación y contiene toda la implementación del Proyecto llevada a cabo haciendo uso del patrón MVC (Modelo-Vista-Controlador) en el lado del servidor. En su interior se encuentran las carpetas fundamentales para la lógica de negocio:
    • “app/Http/Controllers”: Contiene los controladores que responden a las solicitudes del usuario, invocando la lógica del modelo y devolviendo respuestas a la interfaz.
      • StripeController.php: Gestiona la pasarela de pagos y la confirmación de pedidos.
      • TideController.php: Implementa la integración con servicios externos de meteorología y mareas.
    • “app/Models”: Contiene la representación específica de la información con la cual el sistema trabaja (Productos, Pedidos, Usuarios).
    • “app/Http/Middleware”: Contiene filtros de seguridad como AdminMiddleware.php para restringir el acceso al panel administrativo.
Ilustración 8.1.2: Detalle de la carpeta app y lógica MVC
  • La carpeta “resources” contiene el código fuente del frontend. Dado que el sistema utiliza una arquitectura SPA (Single Page Application), esta carpeta es vital para la presentación visual:
    • “resources/js/views”: Contiene las páginas del sistema desarrolladas en Vue.js (Tienda, Carrito, Mareas, etc.).
    • “resources/js/stores”: Almacena la gestión de estado reactivo mediante Pinia (gestión del carrito de compras y sesión de usuario).
    • “resources/js/components”: Contiene elementos de interfaz reutilizables como notificaciones y selectores.
Ilustración 8.1.3: Estructura del frontend basado en Vue.js 3
  • La carpeta “public” contiene el punto de entrada por defecto a la aplicación index.php. Además, actúa como repositorio de recursos estáticos:
    • “public/imagenes”: Almacena las fotografías de las especies locales de Lanzarote y el catálogo de productos de pesca.
    • En esta carpeta también se encuentran los archivos compilados de CSS y Javascript generados por Vite para la producción.
  • La carpeta “database” gestiona la persistencia de los datos:
    • “database/migrations”: Define la estructura de las tablas de la base de datos de forma versionada.
    • “database/seeders”: Contiene los scripts encargados de poblar el catálogo de especies y productos iniciales.
  • La carpeta “routes” contiene las definiciones de los puntos de acceso:
    • api.php: Define los endpoints que el frontend consume para interactuar con la lógica del servidor.
    • web.php: Gestiona la carga inicial de la aplicación y las rutas de autenticación.
Ilustración 8.1.4: Rutas y configuración de la API REST

Finalmente, existen ficheros de configuración clave en la raíz del proyecto:

  • .env: Contiene los parámetros de configuración del sistema, como la conexión a la base de datos y las claves secretas de Stripe.
  • composer.json y package.json: Gestionan las dependencias del backend (PHP) y del frontend (Javascript) respectivamente.
  • vite.config.js: Es el motor de compilación que permite integrar de forma eficiente Vue.js con la estructura de Laravel.
