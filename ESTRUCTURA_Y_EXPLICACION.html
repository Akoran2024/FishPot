Estructura General y Funcional del Proyecto

A continuación se describe la organización jerárquica del proyecto FishPot, detallando el propósito funcional de cada directorio principal dentro de la arquitectura del sistema.

app/

Es el directorio principal del backend. Contiene toda la lógica de negocio, los controladores que gestionan las peticiones, los modelos que interactúan con la base de datos y los middlewares de seguridad.

bootstrap/

Contiene los archivos necesarios para el arranque (bootstrapping) del framework Laravel y la configuración de la caché de servicios para optimizar el rendimiento.

config/

Almacena todos los archivos de configuración global del sistema, como los parámetros de autenticación, la conexión a la base de datos, el sistema de correos y los servicios de terceros como Stripe.

database/

Gestión de la persistencia de datos. Incluye las migraciones para crear las tablas, los seeders para cargar el catálogo inicial de productos y las factorías para generar datos de prueba.

public/

Es el único directorio accesible públicamente desde la web. Contiene el punto de entrada index.php, las imágenes de los productos y especies, y los archivos CSS/JS ya compilados para producción.

resources/

Contiene los archivos fuente del frontend. Aquí se encuentra todo el código de Vue.js 3 (vistas, componentes y estados de Pinia) y las plantillas Blade del servidor.

routes/

Define todos los puntos de acceso o rutas del sistema. Se divide en rutas web para la interfaz de usuario y rutas API para la comunicación de datos entre el frontend y el backend.

storage/

Directorio de almacenamiento interno. Aquí se guardan los logs del sistema, las sesiones de usuario y los documentos generados dinámicamente, como las facturas en PDF.

tests/

Contiene las pruebas automatizadas del sistema, incluyendo pruebas unitarias para lógica aislada y pruebas de integración para flujos completos como el proceso de pago.
