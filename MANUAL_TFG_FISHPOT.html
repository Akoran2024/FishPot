12.11 Anexo I: Manual de Instalación Técnica

FishPot es una aplicación multiplataforma que utiliza Laravel 12 en el lado del servidor y Vue.js 3 para una interfaz de usuario reactiva y fluida. Este manual guía al técnico en la configuración completa del sistema.

12.11.1 Requisitos del Entorno de Desarrollo

Para la ejecución del proyecto es indispensable contar con las siguientes herramientas:

  • PHP 8.2+: Motor de ejecución de la lógica de servidor.
  • Composer: Gestor de paquetes de PHP para Laravel.
  • Node.js 18+ & NPM: Necesarios para la compilación de la interfaz con Vite.
  • MySQL 8.0: Sistema Gestor de Bases de Datos para la persistencia de productos, pedidos y usuarios.

12.11.2 Procedimiento de Despliegue

Siga estas instrucciones en orden secuencial:

1. Preparación del Código Fuente

git clone https://github.com/usuario/FishPot.git
cd FishPot

2. Configuración de Dependencias

Instale las librerías necesarias tanto de backend como de frontend:

composer install
npm install

3. Entorno y Base de Datos

Cree el archivo de configuración y la base de datos:

cp .env.example .env
php artisan key:generate
php artisan migrate --seed
Importante: El comando --seed es vital, ya que carga el catálogo inicial de productos y las categorías necesarias para el funcionamiento del sistema.

12.11.3 Integración de Servicios Externos

FishPot depende de dos servicios críticos que deben configurarse en el archivo .env:

Servicio Variable .env Función
WorldTides WORLD_TIDES_API_KEY Provee los datos de pleamar/bajamar en tiempo real para Lanzarote.
Stripe STRIPE_SECRET Gestiona los cobros seguros mediante tarjeta de crédito.

12.12 Anexo J: Manual de Usuario y Funcionalidades

12.12.1 Módulo de Mareas: Interpretación de Datos

La sección de mareas es la herramienta fundamental de planificación para el pescador. El sistema muestra una tabla detallada con:

  • Hora exacta: Momento preciso del pico de marea (Pleamar/Bajamar).
  • Tipo de marea: Indicador visual de si la marea está subiendo o bajando.
  • Altura: Altura en metros sobre el nivel cero.

Filtrado Inteligente

Gestión de Carrito

12.12.5 Pasarela de Pago Stripe

El proceso de pago es el punto más crítico del sistema. FishPot utiliza Stripe Checkout para garantizar que los datos de pago nunca toquen nuestros servidores, ofreciendo la máxima seguridad al cliente final.

  1. El usuario confirma su carrito.
  2. Se genera una sesión segura en Stripe.
  3. Tras el pago, Stripe notifica al sistema (Webhook/Redirección).
  4. Se genera la orden y se descuenta el stock.

12.12.6 Administración de Contenidos

El administrador dispone de un panel exclusivo para:

  • Gestión de Productos: Subir fotos, editar precios y descripciones.
  • Control de Usuarios: Gestionar permisos y roles de administración.
  • Monitorización de Pedidos: Cambiar estados de "Pendiente" a "Enviado" para notificar al cliente.

13 Glosario Técnico y Académico

Active Record (Eloquent)
Patrón de arquitectura que permite mapear tablas de base de datos a objetos de programación, facilitando las consultas.
API (Application Programming Interface)
Conjunto de reglas que permite que FishPot se comunique con servicios como WorldTides o Stripe.
Lanzarote Fishing Regions
Zonificación de la costa de Lanzarote utilizada en el sistema para categorizar los puntos de pesca (Norte, Sur, Este, Oeste).
PCI-DSS Compliance
Estándar de seguridad para el manejo de tarjetas de crédito, garantizado en FishPot mediante el uso de Stripe.
Seeding
Proceso de poblar la base de datos con información inicial necesaria para que la aplicación no esté vacía al instalarse.
Talla Mínima Legal
Medida mínima establecida por la normativa de pesca de Canarias que debe tener un ejemplar para poder ser capturado.
Webhook
Mecanismo que permite a Stripe enviar una señal a FishPot cuando un pago se ha completado correctamente.
