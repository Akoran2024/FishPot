Anexo: Catálogo Exhaustivo de Ficheros del Sistema

Este documento constituye una descripción técnica pormenorizada de todos los archivos que conforman la aplicación FishPot. La estructura está organizada por capas arquitectónicas para facilitar su comprensión en el marco de la memoria del Trabajo de Fin de Grado.

1. Directorio Raíz: Configuración y Entorno

.env Fichero de configuración de variables de entorno. Almacena las claves secretas de Stripe, credenciales de la base de datos MySQL, claves de la API de WorldTides y parámetros del servidor de correo.
composer.json Archivo de gestión de dependencias de PHP. Define las librerías necesarias como Laravel Framework, Stripe SDK y DOMPDF.
package.json Gestión de dependencias de Node.js. Incluye Vue 3, Pinia, Tailwind CSS y Vite para la compilación del frontend.
vite.config.js Configuración del motor de empaquetado Vite, optimizado para la integración de Vue.js con la estructura de directorios de Laravel.
docker-compose.yml y Dockerfile Definición de la infraestructura contenedorizada. Orquestan los servicios de PHP 8.2, Nginx y MySQL para garantizar la portabilidad del sistema.

2. Lógica del Servidor (Carpeta app/)

2.1 Controladores (app/Http/Controllers)

StripeController.phpManeja el flujo transaccional con la pasarela de pagos, la creación de sesiones y el procesamiento de pedidos confirmados.
TideController.phpGestiona la comunicación con APIs externas para la obtención de datos de mareas de Lanzarote.
ProductController.phpLógica para el listado, filtrado y gestión CRUD de productos y especies marinas.
OrderController.phpGestión administrativa de los pedidos y sus estados de envío.
ClientController.phpAdministra la información de perfiles y direcciones de facturación de los clientes.
Admin/UserController.phpControlador específico para la gestión de usuarios y roles desde el panel de administración.
Auth/...Conjunto de controladores (Login, Register, Password) que implementan el sistema de autenticación Breeze.

2.2 Modelos de Datos (app/Models)

Product.phpModelo de la tabla productos. Incluye lógica de negocio para stock y precios.
Order.phpEntidad principal de pedido, relaciona usuarios con transacciones.
OrderItem.phpDetalle individual de cada producto dentro de un pedido.
User.phpModelo de usuario con gestión de roles (Admin/User).
Client.phpPerfil extendido del usuario para datos de envío.

2.3 Seguridad y Validación (app/Http/Middleware y Requests)

Middleware/AdminMiddleware.phpFiltro de seguridad que restringe el acceso a rutas administrativas.
Requests/OrderStoreRequest.phpValidación estricta de los datos de un pedido antes de procesarlo.
Requests/ProductStoreRequest.phpValidación de campos obligatorios, imágenes y tipos de datos para nuevos productos.

3. Base de Datos (Carpeta database/)

migrations/xxxx_create_users_table.phpEstructura base de la tabla de usuarios y autenticación.
migrations/xxxx_create_products_table.phpDefinición de campos técnicos del catálogo (precios, imágenes, stock).
migrations/xxxx_create_orders_table.phpEsquema para la persistencia de pedidos y transacciones.
seeders/ProductSeeder.phpGuion de carga masiva de datos reales para especies y productos de pesca.
factories/UserFactory.phpGenerador de datos aleatorios para pruebas de carga de usuarios.

4. Frontend y Presentación (Carpeta resources/)

4.1 Vistas Vue (resources/js/views)

Home.vuePágina de aterrizaje con información destacada y accesos directos.
Shop.vueCatálogo reactivo con filtros dinámicos de productos.
Cart.vueInterfaz de gestión del carrito y checkout.
Tides.vueVisualizador de mareas y datos meteorológicos de Lanzarote.
Species.vueEnciclopedia visual de especies marinas locales.
MyOrders.vueHistorial de pedidos y descarga de facturas para el cliente.
Admin/Products.vuePanel administrativo para la gestión CRUD de inventario.

4.2 Lógica Frontend (resources/js/stores y router)

stores/cart.jsEstado global del carrito con Pinia, gestionando la persistencia en local.
stores/auth.jsGestión de la sesión, roles y permisos del usuario en el frontend.
router/index.jsDefinición de todas las rutas de la SPA y control de acceso (Guards).

4.3 Plantillas Blade (resources/views)

app.blade.phpShell principal de la aplicación donde se monta la SPA de Vue.
pdf/invoice.blade.phpPlantilla HTML para la generación dinámica de facturas en PDF.
emails/invoice.blade.phpDiseño del correo electrónico que acompaña al envío de la factura.

5. Definición de Rutas (Carpeta routes/)

api.phpDefine los endpoints de la API REST que consume el cliente Vue.
web.phpGestiona la carga de la aplicación y las rutas de autenticación inicial.
