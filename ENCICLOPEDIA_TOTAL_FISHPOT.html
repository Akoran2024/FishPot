Vol 1: Arquitectura y Sistema

FishPot es una aplicación de tipo Single Page Application (SPA) que utiliza un motor de backend robusto y un frontend reactivo moderno.

Stack Tecnológico

Laravel 12.0 Vue.js 3.5 Tailwind CSS Vite 7.0 Stripe API

Estructura de Ficheros Analizada

app/
├── Http/Controllers/      # Lógica de negocio (Stripe, Tides, Orders)
├── Models/               # Eloquent Models (User, Product, Order)
├── Mail/                # Notificaciones de factura por email
resources/js/
├── views/               # Páginas Vue (Home, Admin, Checkout)
├── router/              # Navegación SPA
├── stores/              # Estado global (Carrito, Auth)
database/
├── migrations/          # Definición de tablas SQL
└── seeders/             # Datos iniciales de productos y admins

Vol 2: Base de Datos y Modelo ER

El sistema utiliza una base de datos relacional (MySQL/MariaDB) con las siguientes entidades principales:

Esquema de Tablas

Tabla Campos Clave Descripción
users id, name, email, password, role Usuarios del sistema (Admin/User).
products id, name, description, price, stock, image Catálogo de artículos de pesca.
orders id, user_id, total, status, payment_intent Cabecera de pedidos y pagos.
order_items id, order_id, product_id, quantity, price Detalle de productos en cada pedido.
Lógica de Stock: El sistema implementa un disparador de decremento en StripeController::success que reduce el stock de la tabla products solo tras la confirmación exitosa del pago.

Vol 3: Backend & API

Controladores Críticos

  • OrderController: Gestiona la creación de pedidos y la generación de facturas PDF usando laravel-dompdf.
  • TideController: Interactúa con la API WorldTides para obtener datos de Lanzarote. Si falla la API, tiene un sistema de Fallback con datos simulados dinámicos.
  • StripeController: Maneja sesiones de Checkout y Webhooks de éxito.

Autenticación y Seguridad

Se utiliza Laravel Breeze para la autenticación base. Se han añadido middlewares personalizados:

// app/Http/Middleware/AdminMiddleware.php
public function handle($request, Closure $next) {
    if (Auth::check() && Auth::user()->role === 'admin') {
        return $next($request);
    }
    return redirect('/');
}

Vol 4: Frontend (Vue.js 3)

El frontend es una aplicación reactiva que se comunica con el backend mediante Axios.

Arquitectura de Componentes

La vista principal está en resources/views/app.blade.php, la cual carga el punto de entrada App.vue.

Gestión de Estado (Pinia)

Se utilizan dos "Stores" principales:

  • authStore: Mantiene la información del usuario logueado y sus permisos.
  • cartStore: Muestra los productos seleccionados, calcula totales en tiempo real y persiste datos en el localStorage para no perder el carrito al refrescar.

Vol 5: Integraciones de Terceros

Pasarela de Pagos Stripe

FishPot implementa Stripe Checkout para máxima seguridad. El flujo es:

  1. El usuario pulsa "Pagar".
  2. El backend crea una Checkout Session con los items y metadatos.
  3. Redirección a Stripe.com.
  4. Tras el pago, Stripe redirige a /stripe/success.
  5. El servidor verifica el payment_intent y crea el registro oficial del pedido.

API de Mareas (World Tides)

Configurada para las coordenadas exactas de Lanzarote:

  • Latitud: 29.035
  • Longitud: -13.633
  • API: v3 (extremes)

Vol 6: Manual de Usuario Completo

Cómo comprar en 5 minutos:

1. Regístrate con un email válido.
2. Añade productos al carrito (fíjate en el stock disponible).
3. Ve al carrito y pulsa "Tramitar Pedido".
4. Introduce tu dirección. El sistema calculará gastos de envío si estás fuera de Canarias (9.50€).
5. Paga de forma segura. Descarga tu factura PDF desde "Mis Pedidos".

Vol 7: Manual del Administrador

ADVERTENCIA: Las acciones en el panel de administración son irreversibles en la base de datos de producción.

Tareas diarias del Admin:

  • Revisión de Ventas: Comprobar los pedidos "Aceptados" (pagados) para su preparación.
  • Control de Stock: Si el stock de un producto llega a 5, aparecerá una alerta visual en el panel.
  • Gestión de Usuarios: Supervisar altas y bajas, y asignar roles a nuevos empleados.
  • Actualización de Precios: Cambiar precios según temporada desde el catálogo administrativo.
