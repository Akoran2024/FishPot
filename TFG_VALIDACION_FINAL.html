8. VALIDACIÓN Y PRUEBAS

Esta fase tiene como finalidad garantizar la calidad del software construido mediante la ejecución de una serie de casos de prueba diseñados para validar la lógica de negocio y la integridad del sistema FishPot.

8.1 Herramientas de Validación

1. PHPUnit

Es el entorno estándar para la realización de pruebas unitarias y de integración en PHP, integrado nativamente en el framework Laravel. Se ha utilizado para:

  • Pruebas Unitarias: Validación de modelos y componentes aislados de la lógica.
  • Feature Tests: Verificación de flujos completos, incluyendo la respuesta de controladores, validación de formularios y persistencia en la base de datos.

2. Laravel Dusk

Herramienta de automatización que simula el comportamiento de un usuario real en un navegador Chrome. Se emplea para testear la interfaz de usuario (UI) y flujos críticos de extremo a extremo (E2E), asegurando que los componentes interactivos de Vue.js funcionen correctamente.

3. Laravel Pint

Utilizado para el control de calidad del código fuente. Asegura que todo el desarrollo PHP cumple con los estándares de estilo PSR-12, garantizando un código limpio, legible y mantenible.

8.2 Ejecución de Pruebas

Para asegurar la integridad del código, se han establecido los siguientes procedimientos de ejecución:

  • Pruebas Funcionales: Ejecutadas mediante el comando php artisan test. Este comando automatiza la creación de una base de datos en memoria (SQLite), ejecuta las migraciones y lanza todos los casos de prueba definidos.
  • Pruebas de Estilo: Se ejecutan con ./vendor/bin/pint para corregir automáticamente desviaciones en el estándar de codificación.
  • Depuración: Uso de php artisan pail para capturar excepciones en tiempo real durante las pruebas manuales en el entorno XAMPP.

Análisis Técnico de Fallos

Los fallos registrados en las suites de Authentication y Profile se deben principalmente a:

  • Falta de Token CSRF (Error 419): Común en pruebas de integración que no simulan correctamente el middleware de sesión de Laravel.
  • Restricciones de Acceso (Error 403): El sistema de roles implementado bloquea el acceso a rutas protegidas si el usuario simulado en el test no posee los privilegios adecuados.
